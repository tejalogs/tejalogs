## What I Do

I build small products that solve one clear problem end to end.

I start with user confusion, design the workflow, write the copy, and ship the frontend.  
Marketing is not a separate phase for me. Distribution is part of the product.

I care about building things people return to, not things that only look good on launch day.

If you are building early-stage products and want someone who thinks in users first and systems second, we will get along.
